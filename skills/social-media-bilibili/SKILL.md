# Social Media - Bilibili

这是 B站平台专属 skill。

目标：
- 把分散的 B站能力收口成一个统一入口
- 只保留可复用、可维护、可解释的能力块
- 优先沉淀稳定链路，避免后续每次从零拼装外部 skill
- 让 `social-media-manager` 可以把 B站相关任务直接路由到这里

## 默认原则

- **公开数据优先 API / 页面公开信息**，不要先上重型浏览器链路。
- **账号写操作再走登录态**，如投稿、动态、私信、点赞等。
- **先复用稳定底盘，再局部补实现**；不要照搬外部 skill 的脏说明和低质流程。
- **成功判据必须是平台真实结果**，不是“按钮点到了”。
- **涉及 cookie / 登录态 / 敏感凭据时，默认按高风险处理。**

## 本次收口策略

本 skill 不是简单复制外部 skill，而是按“去其糟粕，取其精华”重组：

### 已吸收的外部能力来源
- `bilibili-all-in-one`
  - 作为能力地图参考：热门 / 下载 / 观看数据 / 字幕 / 弹幕 / 投稿
- `bilibili-messager`
  - 作为私信浏览器自动化思路参考
- `bilibili-analytics`
  - 作为搜索抓取与统计报告思路参考
- `bilibili-transcript`
  - 作为转录工作流参考

### 不直接继承的内容
- `bilibili-video-publish`
  - 仅保留“浏览器投稿”这个方向，不继承原文档中的脏文案与低质说明
- `bilibili-skill`
  - 仅保留“动态 / 搜索 / 视频管理”的能力范围，不直接继承其混杂实现约定与外部路径假设

## 能力分层

面对 B站任务，优先先判断属于哪一层：

### 1. 内容采集
适合：
- 获取视频信息
- 下载视频 / 音频
- 拉字幕
- 拉弹幕
- 获取分P / 播放地址

### 2. 运营动作
适合：
- 投稿视频
- 发动态

- 账号内互动动作

### 3. 分析监控
适合：
- 热榜 / 趋势观察
- 搜索关键词分析
- UP 主更新跟踪
- 视频数据对比

### 4. 内容处理
适合：
- 视频转录
- 摘要

- 热门/榜单获取

- 字幕下载与处理
- 弹幕获取

- 搜索结果抓取与简单统计
- 视频转录与摘要

### 当前建议谨慎处理的能力

- 点赞 / 互动类账号动作
- 大规模批量抓取

这些功能不是不能做，而是默认要先确认登录态、限频、风控与成功判据。

## Workflow 1：视频信息 / 播放数据获取

目标：获取单个视频的基础信息、UP 主、播放量、点赞、收藏、分P 等公开信息。

### 输入
- B站视频 URL
- 或 BV 号

### 推荐路径
1. 解析 BV 号 / 视频链接
2. 获取视频公开信息
3. 提取：标题、简介、UP 主、时长、封面、统计数据、分P 列表
4. 需要时补拉当前播放 / 点赞 / 评论等数据

### 成功判据
- 返回结构化字段
- 链接 / BV 可复用到后续下载、字幕、分析、转录流程

### 适用场景
- “这个 B站视频讲什么？”
- “给我这个视频的基础数据”
- “看看这个视频的分P”

---

## Workflow 2：字幕 / 弹幕获取

目标：提取 B站视频的字幕或弹幕，用于总结、二次分析、舆情观察。

### 推荐路径

#### 字幕优先级
1. 优先拿官方/公开 CC 字幕
2. 若无可用字幕，再考虑 AI 字幕 / 自动字幕
3. 若仍无，则转入“转录流程”

#### 弹幕
1. 获取视频 cid / 分P 信息
2. 拉取弹幕文本或分段弹幕
3. 需要时做关键词、情绪、梗点分析

### 成功判据
- 成功产出字幕文件或结构化字幕文本
- 或成功产出弹幕列表 / 摘要分析

### 适用场景
- “把这个 B站视频字幕拉下来”
- “分析一下这条视频的弹幕”
- “给我总结这条视频在讲什么”

---

## Workflow 3：视频下载 / 音频提取

目标：将 B站内容保存到本地，用于归档、转录、剪辑、二次处理。

### 推荐路径
1. 先获取视频信息与可用格式
2. 明确目标是：
   - 视频 mp4
   - 音频 mp3
   - 多视频批量下载
3. 下载到本地受控目录
4. 若音视频分离，再本地合并
5. 校验文件存在、大小正常、可被后续流程消费

### 成功判据
- 本地产出文件存在
- 文件名规范、可复用
- 后续可直接进入摘要 / 转录 / 分发链路

### 注意事项
- 高质量格式可能需要登录态
- 合并音视频通常依赖 ffmpeg
- 批量下载要控速，避免风控

## Workflow 4：投稿发布

目标：把本地视频真实投稿到 B站。

### 默认路径选择
优先顺序：
1. **若 API/SDK 链路稳定且登录态完整**：优先程序化投稿
2. **若 API 写操作不稳定或账号链路不明**：退回浏览器创作者中心投稿

### 发布前最小检查
- 本地视频存在
- 标题已准备（<= 80 字）
- 简介已准备
- 标签已准备（建议 <= 7）
- 分区已确认
- 封面已准备或可从视频抽帧
- 已确认 B站登录态可用

### 成功判据
只认以下任一真实结果：
- 稿件管理中可见新稿件
- 返回明确的投稿成功结果与稿件标识
- 页面真实进入投稿成功 / 稿件管理结果页

### 不接受的伪成功
- 仅点到“发布”按钮
- 页面短暂跳转但无稿件结果
- 无法在稿件管理回查到对应作品

### 当前策略
- 浏览器链路可作为保底路径
- 但不要继承外部 skill 中不专业的固定文案或脏模板
- 后续应把已验证成功的真实页面路径沉淀到 `references/`

## Workflow 5：私信互动

目标：给 B站用户发送私信、查看聊天记录、做站内轻运营。

### 前提
- 浏览器已登录 B站
- 已知目标用户名，或可在消息列表中定位到目标会话

### 推荐路径
1. 打开 B站消息页
2. 只在必要时抓快照
3. 定位目标用户会话
4. 进入对话页
5. 以最小注入方式写入消息并发送
6. 发送后回查消息记录是否真实出现

### 成功判据
- 聊天记录中出现新消息
- 或已成功提取到所需历史消息

### 注意事项
- 页面 DOM 可能频繁变化，不要硬编码哈希 class
- 优先基于页面布局、文本和相对位置判断
- 发送频率要克制，避免像脚本刷屏

## Workflow 6：搜索分析 / 趋势观察

目标：围绕关键词、主题、UP 主或视频集合做轻量数据分析。

### 推荐路径
1. 搜索目标关键词
2. 抽取前 N 页结果
3. 统计：
   - 标题关键词
   - 作者分布
   - 发布时间分布
   - 播放 / 评论区间
4. 输出轻量报告

### 适合场景
- “分析 B站上关于 XX 的内容”
- “做一个 B站关键词搜索简报”
- “看看最近这个主题在 B站热不热”

### 产出建议
优先输出：
- 样本范围
- 关键发现
- 值得看的视频 / UP 主
- 下一步建议

---

## Workflow 7：视频转录 / 摘要

目标：把 B站视频内容转成文本，并进一步总结成可消费信息。

### 推荐路径
1. 优先尝试字幕
2. 若没有可用字幕，再走转录
3. 转录完成后，生成：
   - 摘要
   - 要点列表
   - 结构化笔记
4. 需要时把结果保存为 txt / md

### 适用场景
- “总结这个 B站视频”
- “把这条课件视频转成笔记”
- “提炼这条视频的重点”

### 注意事项
- 转录属于内容处理，不是媒体管理主链路的默认第一选择
- 只有在字幕缺失或内容理解需求明确时再启用

## 执行检查清单

执行任一 B站动作前，先快速确认：

- 这是公开信息读取，还是账号写操作？
- 是否需要登录态？
- 输入是 URL、BV 号、关键词、用户名，还是本地文件？
- 成功判据是什么？
- 是否需要下载到本地？
- 是否需要进一步发送、汇报、归档？
